# 🚀 SaSLoop Migration Guide (Oracle to New Server)

Follow these steps when you are ready to move your app to a new server or a custom domain.

## 1. Exporting your Data (Old Server)
On your current Oracle server, run this to save all your customers, orders, and products:
```bash
mysqldump -u root -p sasloop > sasloop_backup.sql
```
*Also, download the `backend/uploads` folder—this contains your product images.*

## 2. Setting up the New Server
1. Install Node.js (v18+) and MySQL.
2. Clone your GitHub repository.
3. Create a new database: `CREATE DATABASE sasloop;`
4. Import your data:
   ```bash
   mysql -u root -p sasloop < sasloop_backup.sql
   ```

## 3. Moving to a Custom Domain (Nginx)
Instead of ngrok, you should use **Nginx** on your new server.
1. Point your domain (e.g., `app.yourbusiness.com`) to your server's IP address.
2. Install Certbot for a free SSL certificate:
   ```bash
   sudo apt install certbot python3-certbot-nginx
   sudo certbot --nginx -d app.yourbusiness.com
   ```

## 4. Updating Meta (WhatsApp)
Go to [developers.facebook.com](https://developers.facebook.com):
1. Find your WhatsApp product.
2. Update the **Configuration -> Webhook URL** to:
   `https://app.yourbusiness.com/api/public/whatsapp/webhook`
3. Verify the token (it should still be `sasloop_verify_token_123`).

## 5. Rebuilding the Frontend
Update your `.env` file on the new server with the new URL and run:
```bash
cd SaSLoop-dashboard
npm run build
pm2 restart all
```
